# Playtest Synthesis: 2026-04-14

## 1. Executive Summary

Three skirmish-tier runs were analyzed, all at Ember Pass, all survived. Characters were all Mist-Bound class due to similar SHA-seeded rolls. Zero deaths means permadeath mechanics remain untested. The core loop — dice, narrative, commit discipline — is solid; the main gaps are a missing damage formula, untested NPC encounters, and a minor commit discipline violation in one run. Overall game health: **good foundation, needs stress-testing at higher tiers and with death scenarios.**

## 2. Pattern: Bugs (by frequency)

**Bug: Location file not updated on first entry**
- **Frequency**: 1/3 runs (20260414-1)
- **Severity**: medium
- **Evidence**: "The ember-pass location file was never updated during the skirmish — `state` remains `unexplored` and `first_entered_by` remains `null`." (Run 1)
- **Note**: Runs 2 and 3 correctly updated location state. Isolated inconsistency — likely depends on whether the GM remembers to check the narrative trigger list.
- **Suggested fix**: Add an explicit checklist item in the per-turn commit checklist in `CLAUDE.md` for location state updates on first entry.

**Bug: Turn 5 commit discipline violation**
- **Frequency**: 1/3 runs (20260414-2)
- **Severity**: medium
- **Evidence**: "Turn 5 narration was committed together with the resolution in a single 'skirmish end' commit, rather than having a separate `turn 5:` commit." (Run 2)
- **Suggested fix**: Add explicit instruction in skirmish/expedition/saga command files that the final turn MUST be committed before the resolution commit. Consider a post-turn script check.

**Bug: Roll count self-reporting error**
- **Frequency**: 1/3 runs (20260414-3)
- **Severity**: low
- **Evidence**: "9 total rolls across 5 turns, not 7. The Meta section and Dice section should say 9." (Run 3 raw notes)
- **Note**: Agent-side reporting error, not a GM bug. The rolls themselves were correct.
- **Suggested fix**: N/A — this is a playtest agent issue, not a game issue.

## 3. Pattern: Rule Violations

**Dice format**: No violations. All 3 runs produced correctly formatted `[Attr N + d20=N = N vs DC N]` lines.

**DC ranges**: Compliant. All DCs fell within skirmish 10–13 band.

**Commit discipline**: 1 violation in Run 2 (turn 5 merged with resolution). Runs 1 and 3 were clean.

**Death handling**: Untested across all 3 runs. No character reached HP ≤ 0.

**Loot generation**: Compliant. Skirmish no-floor-rule applied correctly (Run 1 got nothing on d20=1). Runs 2 and 3 generated valid 8-line loot.

**End-of-turn options**: Displayed correctly in all runs. Fixed `行动 · 对话 · 观察` for skirmish per spec.

**Label namespace**: Minor stretch in Run 2 — `talk-wolves` used for a Body-based intimidation check against animals. The `talk-<npc>` namespace assumes verbal dialog with NPCs. Not a hard violation but a gap in the label spec.

## 4. Pattern: Narrative Issues

**No major narrative issues found.** All 3 runs stayed within 3–6 sentence bounds, avoided emoji, avoided mind-reading, and maintained sensory-only prose. This is the strongest area of compliance.

**Minor observations**:
- Run 3 had a loot narrative disconnect: cart searches failed during play, but loot appeared from the cart in resolution. The mechanical loot roll is correct, but the narrative wrapper contradicted in-game events.
- No NPC dialog occurred in any run (wolves don't talk), so `「」` formatting was never tested.

## 5. Pattern: Friction Points

**No damage formula (3/3 runs noted)**
The most-cited friction point. CLAUDE.md specifies DC/roll mechanics but has no system for calculating damage on failed defense rolls. Run 1: GM improvised 4 HP. Run 2: GM improvised 4 HP and 3 HP. Run 3: no damage taken. Without a formula, damage severity is arbitrary and could vary wildly between sessions.

**NPC encounters never triggered (3/3 runs)**
All runs were at Ember Pass, which lists `wandering-herbalist` as an inhabitant. The NPC never appeared in any run. There's no mechanic for determining whether listed NPCs show up — no encounter roll, no probability table.

**Status effects are labels without mechanics (1/3 runs)**
Run 1 applied `wounded` status with no mechanical effect. No rules define what status tags do.

**Loot narrative disconnect (1/3 runs)**
Run 3's resolution loot appeared from a location the player failed to search. The mechanical roll is separate from narrative, creating a jarring moment.

**Label namespace gaps (2/3 runs)**
`save-herbs-distract` (Run 1) and `talk-wolves` (Run 2) both stretch existing label categories. The spec doesn't cover intimidation, environmental manipulation, or non-verbal animal interactions.

## 6. What Works Well

- **Deterministic dice system**: SHA-seeded rolls via `dice.py` produce believable distributions (rolls ranged 1–20 across sessions) and are fully auditable. All 3 reports praised this.
- **Narrative restraint**: The 3–6 sentence limit and sensory-only rule produce tight, atmospheric prose consistently. No run violated tone.
- **Commit-per-turn architecture**: Creates a clean audit trail. SHA rotation ensures fresh entropy each turn. 2/3 runs executed this perfectly.
- **Character creation speed**: SHA-seeded attributes + class rolling + auto-generated origin produces a playable character in under a minute. All 3 reports called this out positively.
- **Loot generation pipeline**: Seed-based item names are evocative ("Ringlet of the Wandering Mind", "Patient Vest of Cold Breath"). 8-line format works.
- **Creative player actions**: The system handles improvised actions gracefully (herb distraction in Run 1, fire-starting in Run 3, cart cover in Run 2).

## 8. Recommended Next Batch

- **Tier coverage**: Run 2-3 expeditions (15 turns) to test mid-length pacing, `/rest` retreat mechanics, and the loot floor rule. At least 1 saga to stress-test chapter saves and boss mechanics.
- **Death scenario**: Force at least 1 run with a low-HP character or high-DC encounter to trigger permadeath flow. The graveyard system is completely untested.
- **NPC dialog**: Use a location with guaranteed NPC presence (not just "possible encounter") to test `「」` formatting and the `known_to`/`recognized` tracking.
- **Location variety**: All 3 runs used Ember Pass. Run at 2-3 different locations to test location-specific triggers and varied enemy types.
- **Anti-cheat**: Include at least 1 run where the agent attempts to re-roll or modify stats, to verify refusal behavior.
- **Batch size**: Increase to 5 runs (2 expedition, 1 saga, 2 skirmish at new locations). The current 3 skirmish-only batch was too homogeneous.
