# Samples — 三个方向的展示厅

这里是三个候选 MVP 方向的**纯静态样例**。所有内容都是手工写好的 Markdown 文件，
**不依赖任何插件实际运行**——目的是让你在 Obsidian 里打开后凭直觉感受"哪一个我会想真的做下去"。

## 怎么看

1. 把 `Game_on_Obsidian/` 整个目录当作 Obsidian vault 打开
2. 进入下面任一子目录，先读 README，再随意翻看里面的文件
3. 注意每个方向**核心动词不同**——不是"哪个更好"，而是"哪个更像你想做的东西"

## 三方向并非互斥

- **方向 3 天然包含方向 1**——冒险战利品就是 Loot 风格 relic（看 `03-expeditions/Loot/`）
- **方向 2 可作为方向 3 的"基地 NPC"**——出门归来时精灵迎接你
- 渐进路径：**先做方向 1 当地基 → 再叠加 2 或 3**

## 共享的设计原语

无论选哪个方向，以下原语都会出现：

### 双层架构（来自 Claude Buddy）
- **骨骼层**：FNV-1a 哈希(种子) → 确定性属性（slot、rarity、species），**永不写盘**或可被同种子重放
- **灵魂层**：LLM 生成的名字、风味文字、人格——**写到磁盘**

### 纯文本即接口（来自 Loot）
- 8 行装备文本是一种通用格式，方向 1 和方向 3 的战利品都用它
- 没有图片、没有数值——留白让玩家/社区脑补

### Obsidian 原生能力
- **YAML frontmatter** 存属性
- **`[[wiki link]]`** 串联实体（角色 → 装备 → 地点）
- **Dataview** 查询代码块（演示：自动生成"我的收藏"页）
